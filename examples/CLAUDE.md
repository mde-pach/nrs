# Examples — Agent Guide

Two sibling example sets. They serve different purposes and have different editing rules.

## `hello-world/`

A full Next.js project that uses NRS end-to-end. Every layer is populated (`corporate.context.md`, `team.context.md`, `nrs.context.md`, `project.context.md`, plus `domain.context.md` and `implementation.context.md` under `src/domains/*` and `src/app/api/*`). The `CLAUDE.md` in this subtree carries the `DO NOT EDIT — generated by NRS` header — **it is generated**.

**Editing flow:**
1. Change the relevant `*.context.md` file(s).
2. Run `nrs generate claude --dir examples/hello-world` from the repo root.
3. Run `nrs validate --dir examples/hello-world --strict` to confirm the result.
4. Commit the source + generated output together.

Never hand-edit the generated `CLAUDE.md`. If the generator output is wrong, the fix lives in `cli/src/generators/claude.rs` or in the source context files — not in the artifact.

## `formats/`

Standalone one-of-each reference file: `corporate.context.md`, `team.context.md`, `nrs.context.md`, `project.context.md`, `domain.context.md`, `implementation.context.md`. Pure teaching material — not a runnable project.

**Editing rule:** these are canonical examples of what a well-written file at each layer looks like. They must always pass `nrs validate` as if they were real. Keep them tight; every line should demonstrate a rule.

## What Not to Add Here

- No third example duplicating `hello-world` in another framework. If cross-framework coverage is needed, it goes in a dedicated directory and gets justified in `SPEC.md` or `README.md`.
- No scratch projects. `examples/` is published material, not a sandbox.
