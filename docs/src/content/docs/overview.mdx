## The Problem

Without structure, AI tools make developers slower. With structured docs, agents use fewer tokens and finish faster. Irrelevant context hurts reasoning, and removing it improves accuracy.

<Aside type="tip">See [Research](/nrs/research/) for the papers behind these claims.</Aside>

## The Solution

NRS defines what context exists, where it lives, who owns it, and how agents read it. Seven concentric layers, from developer preferences down to source code.

Each layer has a dedicated `*.context.md` file, co-located with what it describes. Generated tool entry points (`CLAUDE.md`, `.cursorrules`, etc.) expose them to agents.

## Core Principles

| Principle | Rule |
|---|---|
| **Density** | No filler, no duplication. |
| **Anti-coupling** | Context survives refactoring. No file paths, no code in domain docs. |
| **Domain-driven** | Each domain described in business terms, not code. |
| **Tests are proof** | No test = unverified spec. Integration over unit. |
| **Propose first** | Agents propose plans, humans approve. |

## The `nrs.context.md` File

Every NRS project starts with an `nrs.context.md` at the root. It tells agents the discovery rules, writing constraints, and testing expectations.

## How the Loop Runs

NRS is a feedback loop: context is authored, loaded, acted on, observed, and refined when gaps surface. See [Lifecycle](/nrs/concepts/lifecycle/) for the stage-by-stage walkthrough and the Claude Code hooks that implement it.
