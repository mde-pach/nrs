## The Problem

Unstructured AI usage **increases** developer completion time. Structured documentation **reduces** agent runtime significantly. Meanwhile, irrelevant context actively **degrades** reasoning — and removing it can **improve** accuracy.

<Aside type="tip">Every claim is backed by peer-reviewed research. See [Research](/nrs/research/).</Aside>

## The Solution

NRS defines **what context exists**, **where it lives**, **who owns it**, and **how it's consumed** — across 7 concentric layers from developer preferences to source code.

Each layer has a dedicated `*.context.md` file, co-located with what it describes. Context files are consumed by agents through generated tool entry points (`CLAUDE.md`, `.cursorrules`, etc.).

## Core Principles

| Principle | Rule |
|---|---|
| **Density** | Every line earns its place. No verbosity, no duplication. |
| **Anti-coupling** | Context survives refactoring. No file paths, no code in domain docs. |
| **Domain-driven** | Clear domain boundaries with business-language context. |
| **Tests are proof** | No test = unverified spec. Integration over unit. |
| **Propose first** | Every action proposed and approved before execution. |

## The `nrs.context.md` File

Every NRS project starts with an `nrs.context.md` at the root. This file tells agents how to work within the NRS system — discovery rules, writing constraints, workflow, and testing philosophy.
